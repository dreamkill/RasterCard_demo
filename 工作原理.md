本文件面向三类读者:工具的使用者、接手改代码的算法工程师、以及被请来帮忙改代码的 AI。
读完应当能回答:这套工具在算什么、为什么这么算、代码怎么组织、每个参数什么含义、想改某个行为该动哪一行、以及标定(pitch test)时有哪些已知的坑。

---

## 0. 一句话概述

把 N 张图(`frames/` 里的帧)按光栅的物理节距交错编织成一张"交织图(interlaced image)",打印后贴上柱透镜光栅,人在不同角度透过透镜看到不同的帧,从而产生 3D / 变图 / 动画效果。另有一个 pitch test 生成器,用来测出"这台打印机 + 这种纸 + 这批光栅"组合下的真实节距。

---

## 1. 术语与符号

| 符号 / 术语 | 含义 | 单位 |
|---|---|---|
| `DPI` / `R` | 打印分辨率,也是切片用的栅格分辨率。本工具里 DPI 和图像 PPI 取同一个值 | dots(pixels)/inch |
| `LPI` | 光栅线数,每英寸多少个柱透镜 | lenses/inch |
| `PPI` | 图像像素密度 = 文件物理尺寸的来源(`物理英寸 = 像素 ÷ PPI`) | pixels/inch |
| `N` | 帧数(`frames/` 里的图片数量) | — |
| `P` | 每个透镜下覆盖的像素数 = `DPI / LPI` | px/lens |
| `strip` | 每帧分到的条带宽度 = `P / N` | px |
| 机械节距 | 厂家标称 LPI(印在包装上的整数,如 50) | lenses/inch |
| 视觉节距 | 实测有效 LPI(如 50.5),受打印机/纸/温湿度影响,**这才是切片要用的值** | lenses/inch |
| 重采样 | 把图像从一种像素密度映射到另一种,会引入插值(轻微变软) | — |

**关键区分:DPI / PPI / 打印机输入分辨率是三个不同的东西**,见 §6。很多混乱都来自把它们当成一个数。

---

## 2. 物理原理(够用就行)

柱透镜光栅是一排半圆柱微透镜。每个透镜把它正下方那一小段像素,按出射角度"分发"到不同方向:你从左看、从右看、或上下倾斜,眼睛接收到的是透镜下不同位置的像素。

所以我们要做的,就是在**每个透镜的宽度内,按顺序塞进 N 帧各一条**。透镜把第 1 条发往一个角度、第 2 条发往另一个角度……于是每个视角看到的是"每个透镜各取同一序号的那一条"拼成的完整一帧。

物理上唯一的硬约束:**纸上一个透镜周期必须正好等于 `1/LPI` 英寸**,编织的条带周期才能和透镜周期对齐。对不齐就串色、鬼影。

## 3. 切片算法(核心)

### 3.1 推导

- 一个透镜的物理宽度 = `1/LPI` 英寸。
- 在分辨率 `R` 下,一个透镜横跨的像素数:`P = R / LPI`。
- 透镜下要塞 `N` 帧,每帧条带宽:`strip = P / N` 像素。
- 输出图第 `x` 列(0 起)落在第几条条带:`s = floor(x / strip) = floor(x · N / P)`。
- 该条带属于第几帧:`frame(x) = s mod N`。

合并:

```
P          = DPI / LPI
strip      = P / N
frame(x)   = floor(x · N / P) mod N
           = floor(x · N · LPI / DPI) mod N
```

**输出第 x 列,整列都取自 `frame(x)` 那一帧的第 x 列。** 帧号随 x 每 `P` 像素循环一次。

### 3.2 为什么用浮点 + floor

`P = DPI/LPI` 通常不是整数(例如 1200/50.5 = 23.76)。如果强行取整,条带宽就会系统性偏移,几个透镜之后就累计错位。用浮点保留小数、对 `x·N/P` 取 floor,误差不累计,只在条带边界产生最多 ±1 像素的取整抖动,这是这种"直接切片"方法的固有特性,肉眼基本不可见。

### 3.3 伪代码

```
输入: frames[N][H][W][C], DPI, LPI
P = DPI / LPI
for x in 0..W-1:
    f = floor(x * N / P) mod N
    输出[:, x, :] = frames[f][:, x, :]
返回 输出[H][W][C]
```

### 3.4 边界情况:条带不足 1 像素

若 `strip = P/N < 1`,意味着每帧分不到一个完整像素,部分帧会被跳过/混叠。
触发条件:`N > P = DPI/LPI`。
对策三选一:**减少帧数 N**、**提高 DPI**、或**降低 LPI**。
代码里 `generate_lenticular.py` 检测到这种情况会打印警告。

### 3.5 竖光栅 vs 横光栅

默认是竖光栅(透镜竖直、左右交织,做横向 3D)。代码里 `interleave()` 按**列**循环。
横光栅(透镜横置)用 `interleave_rows()`,逻辑完全对称,把"列/宽"换成"行/高"。

## 4. Pitch Test 原理(测真实 LPI)

### 4.1 为什么必须测

包装上的 LPI 是四舍五入的标称值。实际视觉节距受打印机真实分辨率、纸张、油墨、甚至温湿度影响,会偏 ±0.5 甚至更多。直接用标称值切片,几乎一定串色。所以正式出图前要测出**视觉节距**。

### 4.2 测试图怎么生成

生成若干横向条带,每条带用一个候选 LPI 画竖直黑白线:

```
对候选 LPI = L_b:
    周期 T = DPI / L_b          # 一个透镜对应的像素数
    像素 x 为黑  当且仅当  (x mod T) < T · duty
```

候选值围绕中心对称展开:`L_b = center + (i - (count-1)/2) · step`,`i = 0..count-1`。

### 4.3 怎么读

打印(**必须 100% 不缩放**,理由见 §5),把光栅贴上去,正对、然后轻轻倾斜。哪一条带看起来**最均匀、整条同色、不跳波纹**,它标的 LPI 就是你这套设备的视觉节距。
先用粗步长(`step 0.1`)找大概,再用细步长(`step 0.01`)围着结果精测一轮。

### 4.4 读数和"测试图的 DPI"绑定

**测试图标签上的 LPI,是在生成测试图时用的那个 DPI 假设下才成立的。** 因为周期 `T = DPI/L_b`。如果你测试用 DPI₁、出正图用 DPI₂,两者不同,50.5 这个读数就不能直接套到正图。
**规则:测试图、正图、以及打印机输入分辨率,三者的 DPI 必须是同一个值。**

## 5. PPI / DPI / 打印缩放 的关系(把"实际大小"讲透)

物理上唯一重要的量是**纸上的物理尺寸(英寸)**,因为透镜对位是按英寸算的。

- `物理英寸 = 像素数 / PPI`。
- 在 PS 里"修改图像大小、**不重采样**":像素一个不动,只改 PPI,也就是只改这堆像素印在纸上多大。
- 打印机有自己的"输入分辨率"(设备栅格)。**只要图像 PPI ≠ 打印机输入分辨率,驱动/RIP 就会重采样**,把图像像素映射到设备点阵上。
- 但"实际大小 / 100% / 不缩放"锁定的是**物理英寸**,不是像素映射。所以:物理尺寸不变 → 透镜对位不变 → 节距不变;变的只是设备像素那一层有没有重采样。

**结论:节距由物理尺寸决定,而"实际大小"把物理尺寸锁死了。打印机那层重采样会让画面轻微变软,但不会改节距。**

### 5.1 两条等效路线

| | 路线 A:点对点 | 路线 B:整数像素 + 缩放 |
|---|---|---|
| 切片每透镜像素 | `DPI/LPI`(小数,如 23.76) | 凑整数(如 24) |
| 文件标的 PPI | = 打印机输入分辨率(如 1200) | `24 × LPI`(如 1212) |
| 打印 | 100% 不缩放,零重采样 | 100%,但 PPI≠设备分辨率 → 全图均匀重采样 ~1% |
| 画质 | 不重采样;条带边有 ±1px 取整抖动 | 一次温和重采样,略软;切片内部是整数 |
| 推荐度 | **推荐**,链路最干净 | 能用,但易忘记缩放、流程脆 |

**本工具走路线 A**:`--dpi 1200 --lpi 50.5` 时,切片直接用 23.76 px/透镜(浮点已处理),输出文件标 1200 ppi,与打印机分辨率一致,点对点打印,完全不用碰 1212,也不用进 PS 改分辨率。

## 6. 一个必须先确认的事实:打印机的"真实输入分辨率"

标称 "1200 dpi" 不一定是打印机吃图的分辨率:

- **喷墨**(爱普生/佳能/惠普):标称 dpi 多指墨点寻址精度。驱动真正吃图的分辨率通常是 **720(爱普生)** 或 **600(佳能/惠普)**。这种情况 `--dpi` 应填 720 或 600,而不是 1200,否则一直在做额外重采样。
- **激光 / CTP / 直接成像**:1200 往往就是输入栅格,可直接 1200 点对点。

**动手前先查清你这台真正的输入分辨率**,测试图和正图都用它,全流程一个 dpi 到底。

### 7.1 关键函数签名(给改代码的人/AI)

`interleave.py`:

```python
pixels_per_lens(dpi: float, lpi: float) -> float
    # 返回 P = dpi/lpi

column_frame_map(width: int, dpi: float, lpi: float,
                 num_frames: int, phase: float = 0.0) -> np.ndarray
    # 返回长度 width 的整型数组,第 x 个元素 = 第 x 列用的帧号(0..N-1)
    # 想换交织规律,改这一个函数就够了

           phase: float = 0.0) -> np.ndarray
    # frames: (N,H,W,C) -> 返回 (H,W,C);列方向交织(竖光栅)

interleave_rows(frames, dpi, lpi, phase=0.0) -> np.ndarray
    # 行方向交织(横光栅)
```

`io_utils.py`:

```python
load_frames(frames_dir, size=(W,H), resample="lanczos") -> np.ndarray  # (N,H,W,3) uint8
```

约定:数组都是 `uint8`,通道顺序 RGB,形状 `(N,H,W,C)` / `(H,W,C)`,`size` 传给 PIL 是 `(宽,高)`。

|---|---|---|---|

## 9. 怎么改 / 怎么扩展(给工程师和 AI)

**改交织规律,只动 `interleave.py::column_frame_map`。** 它把"列号 x → 帧号"的映射独立出来,其余代码不用碰。常见改法:

- **抗摩尔纹的相位微调**:给 `phase` 传非 0 值,平移条带起点。
- **子像素插值(平滑过渡)**:目前是硬切(每列只取一帧)。要让相邻帧在条带边界混合,把 `column_frame_map` 返回的整数帧号换成浮点"帧位置",在 `interleave` 里对相邻两帧做线性插值。这会更柔但略损锐度。
- **条带顺序(正序/倒序/自定义查找表)**:在 `frame(x)` 后面套一个置换表 `order[frame]`,用来纠正某些镜片的左右镜像。
- **RGB 子像素级交织**(用于屏幕类显示如 Looking Glass,而非打印):需要把 R/G/B 三个子像素分别按 1/3 像素偏移映射;这要重写 `column_frame_map` 为按子像素工作,并按光栅倾斜角(slant)加入 y 相关项。
- **倾斜光栅(slanted lenticular)**:帧号同时依赖 x 和 y:`frame(x,y) = floor((x + y·tanθ)·N/P) mod N`。把 `column_frame_map` 升级成接收 `(x,y)` 网格的二维版本。

**改输入输出**,动 `io_utils.py`(如换 16bit TIFF、加 CMYK、改压缩方式)。
**改流程/参数/校验**,动两个 `generate_*.py`。

> 给 AI 的提示:本工具的不变量是"输出第 x 列整列来自单一帧"和"帧号每 P 像素循环"。任何改动若打破这两条(如插值版),要同步更新本文档 §3 和警告逻辑。数组始终 `uint8` / RGB / `(N,H,W,C)`。

## 10. 完整工作流程

1. **查打印机真实输入分辨率**(§6),记为 `R`(喷墨多半 720/600,不是 1200)。
2. **生成测试图**:`python generate_pitch_test.py --dpi R --center-lpi 标称值 --step 0.1 --count 11`。
3. **打印测试图**:100% / 实际大小 / 不缩放,**不要进 PS 改 PPI**。
4. 贴光栅,找最均匀的条带,读 LPI。再用 `--step 0.01 --center-lpi 刚才的值` 精测一轮。
5. **生成正图**:把帧放进 `frames/`,`python generate_lenticular.py --dpi R --lpi 精测值 --width-inch W --height-inch H`。
6. **打印正图**:同样 100% 不缩放。贴光栅检查。

全程一个 `R` 到底,一次缩放都不做。

## 12. 已知陷阱

1. **在 PS 里把图改成非打印机分辨率的 PPI(如 1200→1212)再打印,会污染 pitch test 读数。**
   原因:测试图按 DPI 生成,但你打印时缩放了 `DPI/新PPI`(1200/1212≈0.99,即缩小 1%)。标签写 50.5 的那条,纸上实际印成 `50.5/0.99 ≈ 51.0 LPI`。于是你读到的"50.5"和纸上真实节距差了 1%。
   正图若也用同样的缩放,两个 1% 会互相抵消、碰巧能用——但这是"两个错误相消"的巧合,流程脆,任一步忘记缩放就全乱。
   **干净做法**:测试图和正图都用打印机真实分辨率生成、100% 不缩放打印,读数即真值(本工具默认走这条)。
2. **喷墨的标称 dpi ≠ 输入分辨率**(§6)。按 1200 切片喂给实际吃 720 的喷墨,等于一直在重采样。
3. **节距随环境漂**。温湿度变化会让塑料光栅微胀缩,改变有效 LPI。重要活件在出图前重测一次,或在恒温恒湿环境作业。
4. **打印时缩放 = 前功尽弃**。任何非 100% 的缩放都会改物理节距,让光栅对不上。
5. **大图吃内存**。6×4 英寸 @1200dpi ≈ 7200×4800,N 帧一次性载入。内存不够就降 dpi 测试,或改 `interleave` 为分块处理。
